# Agent Development Standards

## Naming Conventions

- **Agent Names**: Use clear, descriptive names (e.g., "HR Benefits Assistant", not "HR Bot")
- **Topics**: Start with verb (e.g., "Search Knowledge Base", "Create New Request")
- **Actions**: Descriptive of what they do (e.g., "Get Employee Record", "Send Approval Email")

## Description Format

Every agent, topic, and action must have a description following this template:

**Purpose**: What this does and when to use it
**Inputs**: What information is required
**Outputs**: What the user gets
**Limitations**: Any constraints or special considerations

## Required Documentation

Every agent must have:
- README.md with overview and usage instructions
- CHANGELOG.md tracking all changes
- knowledge-sources.md listing all data sources used
- Conversation flow diagram or description

## Code of Conduct

- Test before publishing
- Document all changes
- Complete quality checklist
- Get peer review
- Update Agent Catalog immediately after publishing
